API

Nazwa Wartość domyślna Opis
showDuration 300 Pokazuje animacje przez podaną liczbę milisekund. Wartość 0 wyłącza animacje.
showEasing 'ease' Nadaje animacji efekt: 'ease'. Podajemy jako łańcuch znaków. Inne przykładowe rodzaje animacji: ease-in, ease-out, step-start.. itd
hideDuration 300 Animacja ukrywania przez podaną liczbę milisekund. Wartość 0 oznacza wyłączenie animacji ukrywania.
hideEasing 'ease' Nadaje animacje ukrywania 'ease'. Inne przykładowe rodzaje animacji, które można tu wpisać: ease-in, ease-out, step-start
visibleStyles {opacity: 1, transform: 'scale(1)'} Podane style będą aplikowane do wszystkich widocznych elementów. Te style są także używane do pokazywania/ukrywania animacji.
hiddenStyles {opacity: 0, transform: 'scale(0.5)'} Podane style będą aplikowane do wszystkich niewidocznych elementów. Te style są także używane do pokazywania/ukrywania animacji.
layout {fillGaps: false, horizontal: false, alignRight: false, alignBottom: false} Definiuje w jaki sposób elementy będą układane. Domyślnie, Muuri ma wbudowany domyślny algorytm. Możemy zaimplementować własny algorytm.
layoutOnInit true Ustawia czy Muuri powinien uruchamiać metodę layout podczas inicjalizacji.
layoutDuration 300 Czas animacji układu elementu w milisekundach. Wartość 0 wyłącza layoutDuration.
layoutEasing 'ease' Animacja 'ease' układu elementu. Dopuszczalne są inne prawidłowe rodzaje animacji ease. (link: Animacje ease)
sortData null Podając tej metodzie obiekt gdzie kluczem jest nazwa atrybutu, który można posortować zwraca informacje o tym, który z elementów da się posortować.
dragEnabled false Ustawia czy elementy można 'przeciągać'.
dragContainer null Kontener do którego element powinien być wstawiony na czas przeciągania go. Jeżeli ustawione na null, użyty zostanie kontener siatki
dragStartPredicate {distance: 0, delay: 0, handle: false} Funkcja ustawiająca kiedy element powinien rozpocząć przemieszczanie gdy będzie przeciągany. Domyślnie używa wbudowanego predykatu, który ma kilka wbudowanych opcji.
dragAxis null Ustawia czy elementy mają być przemieszczane poziomo czy pionowo. Jeżeli 'x' to poziome ustawianie. Jeżeli 'y' to pionowe ustawianie. Gdy 'null' - elementy mogą być przemieszczane pionowo i poziomo.
dragSort true Ustawia czy elementy powinny być sortowane podczas przeciągania
dragSortInterval 100 Definiuje ilość czasu przez który element musi być przeciągany aby nastąpiło aktywowanie funkcji dragSortPredicate. Wartość 0 nie jest rekomendowana.
dragSortPredicate {action: 'move', threshold: 50} Definiuje logikę procedury sortowania podczas przeciągania elementu
dragReleaseDuration 300 Czas animacji upuszczania przeciąganego elementu. Ustawienie na 0 spowoduje wyłączenie tej funkcjonalności.
dragReleaseEasing 'ease' Animacja 'ease' przy upuszczaniu przeciąganego elementu. Dostępne animacje ease: animacje ease
containerClass 'muuri' Nazwa kontenera elementu
itemClass 'muuri-item' Nazwa elementu
itemVisibleClass 'muuri-item-shown' Nazwa widocznego elementu klasy
itemHiddenClass 'muuri-item-hidden' Nazwa ukrytego elementu klasy
itemPositioningClass 'muuri-item-positioning' Nazwa tej klasy jest dodawana do elementu na czas pozycjonowania(position)
itemDraggingClass 'muuri-item-dragging' Nazwa tej klasy jest dodawana do elementu na czas przeciągania(drag)
itemReleasingClass 'muuri-item-releasing' Nazwa tej klasy jest dodawana do elementu na czas upuszczania(drop)
grid.getElement() - Zwraca element(jego instancje)
grid.getItems( [targets] ) - Zwraca wszystkie elementy będące w siatce
grid.refreshItems( [items] ) - Odświeża wymiary elementów w siatce. Jeżeli nie podano argumentów, wszystkie elementy zostaną zsnychronizowane.
grid.refreshSortData( [items] ) - Synchronizuje dane posortowanych elementów
grid.synchronize() - Synchronizuje elementy w taki sposób aby były ułożone zgodnie ze strukturą DOM
grid.layout( [instant], [callback] ) - Wylicza pozycje elementów i ich pozycje chyba, że są już ułożone prawidłowo. Wysokość i szerokość siatki w której są elementy jest także dostosowywana do liczby elementów.
grid.add( elements, [options] ) - Dodaje nowe elementy do siatki.
grid.remove( items, [options] ) - Usuwa elementy z instancji grid
grid.show( items, [options] ) - Pokazuje elementy instacji grid
grid.hide( items, [options] ) - Ukrywa podane elementy
grid.filter( predicate, [options] ) - Filtruje elementy siatki grid według zadanych parametrów np. koloru
grid.sort( comparer, [options] ) - Sortuje elementy
grid.move( item, position, [options] ) - Przemieszcza element w inną pozycję na siatce
grid.send( item, grid, position, [options] ) - Przerzuca elementy do innej siatki
grid.on( event, listener ) - Binduje podsłuchiwany event(event - łańcuch, listener - funkcja)
grid.once( event, listener ) - Binduje podsłuchiwany event, który aktywuje się tylko raz
grid.off( event, listener ) - Odbindowuje podsłuchiwany event
grid.destroy( [removeElements] ) - Niszczy instancje grid

Na stronie autora( sprawdź zakładkę odsyłacze ) znajdziesz o wiele więcej funkcji na, które warto zwrócić uwagę. API autora wzbogacone jest także o przykłady i szczegółowe opisanie argumentów, które dana metoda przyjmuje. Zachęcam do odwiedzenia API autora :)
